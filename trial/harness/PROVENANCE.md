# Constants provenance — the full audit ledger

**The rule: a number without its evidence is a bug.** Every constant in the
relocalization / loop-closure / fiducial / mapping stack either has a recorded reason to
be its exact value — a sweep row, a win-table entry, a rationale comment, a measurement —
or it is a guess wearing a decimal point. This ledger is the audit: every tunable
constant the sweep found, with the strongest evidence that exists for its exact value,
and an honest status when none does.

**Counts: arbitrary 49 · partial 30 · tested 29 — 108 unique constants.**

- **tested** — the exact value was swept or bracketed against data; the receipt is cited.
- **partial** — recorded rationale, mechanism-level testing, or after-the-fact
  measurement exists, but the exact value was never independently varied.
- **arbitrary** — no recorded basis anywhere: no sweep row, no comment, no commit
  rationale.
